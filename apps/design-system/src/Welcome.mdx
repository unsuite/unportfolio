# unportfolio — Design System

Vetrina dei design token (`@unportfolio/ui-tokens`) e superficie di revisione
delle decisioni di prodotto. Vedi [ADR-0006 (Design system)](../../../docs/adr/0006-design-system.md)
e [ADR-0007 (Product & reference)](../../../docs/adr/0007-product-and-reference-spaces.md).

## Com'è organizzato

- **Foundations** — i design token (colori, palette chart, spazi, tipografia)
  generati dalle custom properties CSS di `@unportfolio/ui-tokens`.
- **Components** — il catalogo delle primitive presentazionali di
  `@unportfolio/ui` (`packages/ui`), stilate con CSS Modules sui token (ADR-0006).
  Ogni componente è puro (dati via props, nessun core/store/router/fs) e ha una
  storia per variante reale. Le viste di `apps/web` non sono ancora ricablate su
  questi componenti: il catalogo serve a ragionare su ciascuna primitiva in
  isolamento.
- **Product** — ogni decisione di prodotto (PDR in `docs/product/decisions/`)
  viene renderizzata qui come pagina: si "battezza" il documento dandogli una
  vetrina condivisa dove gli stakeholder rivedono legacy → prototipo → scelta.

## Aggiungere una decisione

Usa la skill `product-decision`: crea il PDR da template e la pagina
`src/Product/<Decisione>.mdx` che lo importa `?raw`, così il markdown resta
l'unica fonte di verità.
